How It Works
VM1/VM2 Architecture:
  1. VM1 (Firewall Gateway): Inspects all traffic with nftables + Suricata DPI
  2. VM2 (This Web Server): Runs vulnerable Flask application
  3. Traffic Flow: Client → VM1 (inspection) → VM2 (processing)
  4. Malware Detection: VM2 scans files with ClamAV, alerts VM1 to block infected uploaders
  5. Adaptive Response: VM1 dynamically blocks malicious IPs based on detected attacks
Note: VM2 never sees real client IPs (only VM1's IP: 10.0.0.1). VM1 correlates malware alerts with its conntrack table to identify and block real attackers.
