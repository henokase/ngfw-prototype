About This Project

Adaptive NGFW Test Website

A deliberately vulnerable web application designed to test Next-Generation Firewall (NGFW) detection, inspection, and adaptive response capabilities.

Architecture
Dual-VM Topology:
  • VM1 (Firewall Gateway): nftables + Suricata DPI
  • VM2 (Web Server): Flask application (this server)
Traffic Flow:
Client → VM1 (Inspection) → VM2 (Processing)

VM2 IP: 10.0.0.5 | VM1 IP: 10.0.0.1

Vulnerabilities
  • CRITICAL SQL Injection
  • CRITICAL Command Injection
  • CRITICAL Unrestricted File Upload
  • HIGH Path Traversal
  • HIGH XSS (Stored & Reflected)
  • HIGH XXE (XML External Entity)
  • MEDIUM Open Redirect
Malware Detection
ClamAV Integration:

All uploaded files are scanned with ClamAV antivirus engine.

Adaptive Response:
  1. VM2 detects infected file
  2. VM2 sends alert to VM1 (filename, hash, signature - NO IP)
  3. VM1 correlates with conntrack table
  4. VM1 dynamically blocks real attacker IP
Note: VM2 never sees real client IPs due to NAT translation by VM1.
Important Warning

FOR EDUCATIONAL AND TESTING PURPOSES ONLY!

This application contains intentional security vulnerabilities. It should:

  • NEVER be deployed to production environments
  • NEVER be exposed to the public internet
  • ONLY be used in controlled lab environments
  • ONLY be used for security testing and education
